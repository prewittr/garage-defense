### Deferred to Editor follow-up (still in RPM-007 scope)

The following work items cannot be authored safely outside the Unity Editor — `.unity` YAML and `AddressableAssetSettings` are too fragile to hand-write. They remain part of RPM-007; the ticket does **not** move to `In Review` until a follow-up commit from a developer with Unity Editor access lands them:

1. **`Scenes/_Boot.unity`** — new Scene containing one `Bootstrap` GameObject with the `Rpm.App.Bootstrap` `LifetimeScope` component attached. Must be the first entry in Build Settings scene list.
2. **`Scenes/TactileTerror_S1.unity`** — empty playable scene loaded async via Addressables after boot completes; satisfies the AC "scene `Scenes/TactileTerror_S1.unity` is loaded via Addressables async."
3. **Addressables initialization** — open `Window → Asset Management → Addressables → Groups`, generate `AddressableAssetSettings`, mark `TactileTerror_S1.unity` as an Addressable entry, and commit the resulting `Assets/AddressableAssetsData/` folder.
4. **Bootstrap GameObject wiring** — ensure the `LifetimeScope` instance in `_Boot.unity` has `AutoRun = true` and its `Parent` reference is null (root scope).
5. **Meta files** — Unity will generate `.meta` files for the new asmdefs, scenes, and Addressables data on first open. Commit those `.meta` files in the same follow-up.

The editor smoke (load `_Boot.unity`, Play, verify Addressables crossfade into `TactileTerror_S1.unity` with no exceptions) is Jasmine's QA gate for this ticket.
