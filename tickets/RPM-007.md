# [RPM-007] Module asmdef Split + VContainer Composition Root

**Feature:** [FEAT-001](FEAT-001-kinetic-door.md) (infrastructure prerequisite)
**Owner:** **Terrell Vaughn** (lead) + **Kendra Brooks** (support)
**Size:** M
**Risk:** 🟢
**Status:** **Ready**

## User Story
**As the** dev team
**I want** the flat `Rpm` assembly split into feature-oriented `.asmdef` modules with a `VContainer`-backed composition root
**So that** Sprint 1 gameplay code lands inside clean module boundaries from day one (instead of becoming technical debt we have to unwind later)

## Acceptance Criteria
- [ ] **Given** the current flat `Rpm` assembly, **when** this ticket lands, **then** the following assemblies exist with declared dependencies matching [ARCHITECTURE.md §2](../docs/architecture/ARCHITECTURE.md):
  - `Rpm.Core` (no deps)
  - `Rpm.Input` (Core + Unity.InputSystem)
  - `Rpm.Gameplay.Door` (Core)
  - `Rpm.Gameplay.Scrap` (Core + Input)
  - `Rpm.Juice` (Core)
  - `Rpm.App` (all of above + UniTask + VContainer)
- [ ] **Given** any two assemblies, **when** the compile-dependency graph is checked, **then** there are **zero circular references**.
- [ ] **Given** a fresh clone + `Unity Hub → open project`, **when** Unity boots, **then** `Rpm.App.Bootstrap` runs and VContainer wires the container with lifetimes per ARCHITECTURE.md §3 (Singleton / Scoped / Transient).
- [ ] **Given** `Rpm.App.Bootstrap` completes, **when** the crossfade ends, **then** scene `Scenes/TactileTerror_S1.unity` is loaded via Addressables async (no sync `SceneManager.LoadScene`).
- [ ] **Given** Editor tests run, **when** CI executes the Unity test runner, **then** a single smoke test (`Rpm.App.Tests.BootstrapSmoke`) passes verifying the container resolves at least one `Singleton` scoped service.
